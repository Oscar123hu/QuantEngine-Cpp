# QuantEngine-Cpp 📈

![Language](https://img.shields.io/badge/Language-C%2B%2B17-blue) ![Build](https://img.shields.io/badge/Build-CMake-orange) ![License](https://img.shields.io/badge/License-MIT-green)

`QuantEngine-Cpp` 是一个基于 **C++17** 标准开发的高性能、低延迟流式量化回测引擎内核。系统实现了从“磁盘大文件读取、内存复用解析、实时指标递推”到“订单簿盘口撮合”的完整高频事件驱动（Event-Driven）流水线，专注于压榨底层 IO 与内存流转效率。

---

## ⚙️ 核心架构与高性能设计

整个引擎由四条硬核防线交织而成，旨在高频/海量历史行情流入时维持极低的延迟和稳定的内存占用：

### 1. 多级盘口快速检索与 FIFO 撮合 (std::map + std::list)
* 放弃全局线性扫描，采用复合拓扑架构：**买盘（Bids）**与**卖盘（Asks）**分别由自平衡红黑树（`std::map`）管理，盘口极速暴露复杂度为 $O(1)$，新价格档位检索控制在 $O(\log N)$。
* 价格档位内部嵌套双向链表（`std::list`），严格遵循 **FIFO（先进先出）** 的时间优先撮合铁律。
* **$O(1)$ 级无痕撤单**：高频交易中撤单频繁，系统支持在持有迭代器指针的情况下，在 $O(1)$ 时间内将任意委托单拔除，规避数组搬移造成的延迟抖动。

### 2. 内存复用型文件流通道 (File I/O Streamer)
* 采用 `std::ifstream` 流式吞噬本地真实 CSV 行情文件。
* 核心性能优化：在数据循环外部声明全局复用缓冲区 `std::string line`。利用 `std::getline` 的内部机制，在读取新行时直接在原地覆盖字符，**彻底杜绝了频繁开辟动态堆内存（Reallocation）的开销**。

### 3. 0 字节堆内存拷贝解析器 (Zero-copy Parser)
* 引入 C++17 `std::string_view`，解析时作为“轻量探照灯”直接映射在复用缓冲区上进行逗号切片，全程不发生任何 `std::string` 堆内存申请。
* 引入 `<charconv>` 中的 **`std::from_chars`** 底层数字转换武器，直接在指定的原始内存区间内硬核解析 double 与 int，速度比传统的 `std::stod` 快数倍且严格做到零内存分配。

### 4. $O(1)$ 空间复杂度流式策略大脑 (Streaming Indicator)
* 摒弃传统的“历史价格大数组批量计算”逻辑，采用**在线流式递推算法（Online Algorithm）**。核心计算公式（以 5 周期 EMA 为例）：
  $$EMA_t = \alpha \cdot Price_t + (1 - \alpha) \cdot EMA_{t-1}$$
* 状态机内部仅保留上一状态变量，无论行情数据流入 10 万条还是 100 亿条，**空间复杂度严格维持在 $O(1)$**，完美保护 CPU L1/L2 缓存，实时计算并动态激发交易信号。

## 🛠️ 构建与运行指南

本项目采用标准的 CMake 构建系统，无任何第三方库依赖，具备极强的平台通用性。

### 1. 编译构建
```powershell
# 创建并进入构建沙盒
cmake -B build

# 执行全量编译
cmake --build build

# Windows 平台
.\build\Debug\QuantEngine.exe

# Linux / macOS 平台
./build/QuantEngine
