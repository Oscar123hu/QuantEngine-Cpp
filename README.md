# QuantEngine-Cpp

基于 C++ 开发的轻量化流式量化撮合与策略回测内核。系统专注于高频 Tick 行情数据的非拷贝流式解析、基于“价格-时间优先”原则的限价订单簿（Limit Order Book）构建，以及流式趋势策略的状态机增量计算。

## 核心技术架构

### 1. 限价订单簿拓扑设计 (Order Book Topology)
* **双向价格排序**：采用 `std::map` 作为红黑树索引，买盘盘口按价格降序排列（`std::greater<double>`），卖盘盘口按价格升序排列（`std::less<double>`），确保最佳盘口价格（买一、卖一）处于 $O(1)$ 的可访问边界。
* **时间优先流转**：每个价格档位内部嵌套 `std::list` 双向链表。新订单追加至链表尾部，撮合阶段从链表头部（最老订单）开始顺序扣减数量，严格遵循先进先出（FIFO）的时间优先原则。

### 2. 零拷贝流式解析流 (Zero-Copy Parser)
* **动态内存零开销**：行情解析层全量基于 C++17 `std::string_view` 实现。在逐行切分 CSV 数据时，系统仅维护指向原始文件缓冲区的只读指针与长度标记，避免频繁在堆（Heap）上申请与销毁 `std::string` 导致的延迟抖动。
* **低级类型转换**：利用 `<charconv>` 库中的 `std::from_chars` 函数替代传统的 `atof` 或 `std::stod`。执行过程无异常抛出开销、与本地化格式（Locale）无关，直接操作原始字符数组完成极速数字解析。

### 3. 增量状态机策略 (State-Machine Strategy)
* **空间复杂度 $O(1)$**：策略类（`SimpleStrategy`）摒弃了缓存历史行情大数组的传统设计，采用状态机递推模型。在运行期仅通过一个浮点数维护上一期 EMA（指数移动平均线）状态。
* **流式顺时激发**：每当一行行情通过总线注入，策略直接触发一次轻量化乘加递推，并顺时完成均线突破信号的捕捉与激发。
